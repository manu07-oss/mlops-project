# MLOps Salary Prediction Pipeline > End-to-end MLOps project — from raw data to a monitored, versioned, production-deployed ML model using industry-standard DevOps + ML tooling. [![CI/CD](https://github.com/manu07-oss/mlops-project/actions/workflows/ci-cd.yaml/badge.svg)](https://github.com/manu07-oss/mlops-project/actions) [![Docker Hub](https://img.shields.io/badge/Docker%20Hub-manognavengala01%2Fsalary--prediction--api-blue)](https://hub.docker.com/r/manognavengala01/salary-prediction-api) --- ## What problem does this solve? In traditional ML, a data scientist trains a model on their laptop and hands over a `.pkl` file. Nobody knows which version is in production, what data it was trained on, how it's performing today, or how to redeploy it when it degrades. This project solves all of that by wrapping an ML model with a full DevOps lifecycle. --- ## Architecture ``` Developer Laptop ├── data/ (versioned) ├── src/train_v2.py → MLflow tracking → 3 model comparison └── src/predict.py → FastAPI /predict endpoint CI/CD Pipeline (GitHub Actions) ├── Trivy code scan (filesystem) ├── Docker build ├── Trivy image scan (CRITICAL + HIGH) └── Push to Docker Hub Production (Kubernetes) ├── 3 replicas of FastAPI service ├── Liveness + readiness health probes ├── LoadBalancer service └── Rolling deployments — zero downtime ``` --- ## Tech Stack | Layer | Tool | Purpose | |-------|------|---------| | ML Training | scikit-learn | LinearRegression, RandomForest, GradientBoosting | | Experiment Tracking | MLflow | Log params, metrics, model artifacts per run | | Model Serving | FastAPI | REST API — POST /predict | | Containerization | Docker | Portable image — runs anywhere | | CI/CD | GitHub Actions | Auto build + scan + push on every git push | | Security Scanning | Trivy | Scan code + Docker image for CVEs | | Orchestration | Kubernetes | 3 replicas, auto-restart, rolling updates | --- ## Model Results | Model | MAE | R2 Score | Result | |-------|-----|---------|--------| | LinearRegression | 16,536 | 0.61 | Baseline | | RandomForest | 16,586 | **0.75** | ✅ Promoted to production | | GradientBoosting | 16,942 | 0.70 | — | RandomForest selected — highest R2 score, best pattern understanding. --- ## API ```bash GET /health → {"status": "ok", "model": "RandomForest"} POST /predict → {"years_experience": 5, "education_level": 3} ← {"predicted_salary": 45860.0, "model_used": "RandomForest"} GET /docs → Interactive Swagger UI ``` --- ## Run locally ```bash git clone https://github.com/manu07-oss/mlops-project.git cd mlops-project python -m venv venv && venv\Scripts\activate pip install -r requirements.txt python src/train_v2.py python src/save_model.py python -m uvicorn src.predict:app --reload ``` --- ## Run with Docker ```bash docker pull manognavengala01/salary-prediction-api:latest docker run -p 8000:8000 manognavengala01/salary-prediction-api:latest ``` --- ## Deploy to Kubernetes ```bash kubectl apply -f k8s/deployment.yaml kubectl apply -f k8s/service.yaml kubectl get pods minikube service salary-prediction-service --url ``` --- ## Real bugs fixed **pywin32 on Linux** — pip freeze captured Windows-only packages. Docker runs Linux. Fixed by filtering requirements.txt before build. **MLflow Windows path in Docker** — MLflow stored absolute D:/ paths in SQLite. Container couldn't find them. Fixed by exporting model to portable pickle file at build time. **Kubernetes liveness probe too aggressive** — Probe checked /health before ML model finished loading. Container restarted in a loop. Fixed by increasing initialDelaySeconds to 60. **Docker layer cache** — Code changes not picked up. Fixed with docker build --no-cache. --- ## Interview one-liner > "I built an end-to-end MLOps pipeline — trained and compared 3 models with MLflow, served the best model as a FastAPI REST API, containerized it with Docker, automated build and security scanning with GitHub Actions and Trivy, and deployed on Kubernetes with 3 replicas and zero-downtime rolling updates." --- ## Author **Manogna** — DevOps Engineer transitioning to MLOps/AI Infrastructure GitHub: [manu07-oss](https://github.com/manu07-oss) | Docker Hub: [manognavengala01](https://hub.docker.com/u/manognavengala01)
